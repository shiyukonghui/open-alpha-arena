关于Alpha Arena架构设计的部分：

1. Alpha Arena是一个让大型语言模型(LLM)在真实加密货币市场中进行交易的平台
2. 模型使用Hyperliquid交易所进行永续合约交易
3. 模型接收数值市场数据作为输入，进行自主交易决策
4. 每个模型的目标是最大化损益(PnL)

从文件中，我可以提取出以下关于架构设计的关键信息：

1. **推理循环设计**：
   - 每2-3分钟进行一次推理调用
   - 模型接收(a)系统提示(指令集)和(b)实时市场+账户状态(用户提示)
   - 模型返回结构化操作，输入到Hyperliquid交易执行管道

2. **输入数据**：
   - 市场数据：当前和历史中间价、成交量、技术指标(EMA、MACD、RSI等)
   - 账户状态：总回报率、可用现金、当前账户价值、持仓表现等
   - 数据按时间顺序排列：最旧→最新
   - 时间框架：日内序列以3分钟间隔提供，还有4小时时间框架数据

3. **输出格式**：
   - 交易信号：buy_to_enter(买入建仓)、sell_to_enter(卖出建仓)、hold(持有)、close(平仓)
   - 交易参数：目标硬币、方向(多头/空头)、数量、杠杆
   - 附加信息：简短理由、[0,1]置信度分数、退出计划(利润目标、止损、失效条件)

4. **头寸管理**：
   - 头寸规模由模型自身计算，基于可用现金、杠杆和风险偏好
   - 退出计划包含明确的利润目标、止损和失效条件
   - 模型可以持有多个头寸(最多6个)，但不同模型有不同的头寸管理策略

5. **框架特点**：
   - 避免多智能体编排、工具使用和长对话历史
   - 为模型提供浓缩的实时市场特征
   - 要求模型提供明确的退出计划
   - 将头寸规模与模型的置信度分数挂钩

6. **技术实现**：
   - 使用永续合约进行交易，支持杠杆
   - 交易品种：BTC、ETH、SOL、BNB、DOGE、XRP六种加密货币
   - 中低频交易(MLFT)，决策间隔为几分钟到几小时

7. **风险管理**：
   - 模型必须处理定量数据并进行系统化交易
   - 每次调用时提供夏普比率，帮助规范风险行为
   - 要求模型设定明确的退出计划，包括利润目标、止损和失效条件

8. **框架限制**：
   - 模型没有显式制度意识
   - 无法访问先前状态-操作历史
   - 不支持金字塔建仓(增加或减少当前头寸)
   - 有限的上下文窗口

从这些信息中，我可以总结出Alpha Arena的架构设计特点：

1. **简洁的输入输出设计**：提供浓缩的市场特征，要求结构化的交易输出
2. **闭环交易系统**：从市场数据输入到交易执行形成闭环
3. **风险管理机制**：通过退出计划、头寸规模控制和杠杆管理来控制风险
4. **中低频交易策略**：适合LLM处理的决策频率
5. **单一模型决策**：不使用多模型协作，避免复杂性
6. **透明度和可审计性**：所有交易和决策可追踪，便于分析
